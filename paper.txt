АННОТАЦИЯ
В данном документе приведено описание программы «название», предназначенной для автоматизации учёта пользователей, то, что указано в таблицах 1-го модуля (например заказчиков, материалов, заказов и продукции.). Исходным языком программы является C#. Среда разработки – Microsoft Visual Studio.
Основной функцией программы является предоставление пользовательского интерфейса для взаимодействия с базой данных и валидации ФИО через стороннее API.
Система реализует следующие функции:
•	Авторизация пользователей с капчей и разграничением прав доступа; 
•	Блокировка пользователя после трёх неудачных попыток ввода пароля или прохождения капчи;
•	Валидация данных через внешнее API;
•	Управление пользователями (для администраторов);
•	Просмотр данных из БД.

1.	ОБЩИЕ СВЕДЕНИЯ
1.1.	Обозначение и наименование программы
•	Наименование исполняемого файла – название.exe;
•	Версия файла – 1.0;
•	Производитель – ФИО;
•	Язык – Русский.
1.2.	Программное обеспечение, необходимое для функционирования программы
•	ОС: Windows 10/11;
•	Среда выполнения: .NET Framework (версия не ниже 10.0.0);
•	СУБД: MySQL;
•	ORM: Entity Framework Core.
1.3.	Языки программирования
•	Язык: C#;
•	Среда разработки: Microsoft Visual Studio. 
2.	ФУНКЦИОНАЛЬНОЕ НАЗНАЧЕНИЕ
2.1.	Классы решаемых задач
•	Обеспечение целостности информации о пользователях, то, что указано в таблицах 1-го модуля (например заказчиков, материалов, заказов и продукции.)
•	Выполнение операций добавления, редактирования и удаления данных (для администратора);
•	Просмотр данных;
•	Аутентификация пользователей с проверкой CAPTCHA;
•	Блокировка учётной записи после трёх неудачных попыток входа;
•	Валидация данных через внешнее API;
2.2.	Назначение программы
Программа работает под управлением Windows и функционирует в среде .NET Framework. Основная функция – предоставление централизованного графического интерфейса для управления ресурсами указанная тема (молочный комбинат, банк, отель и т.д.) с использованием Entity Framework для работы с базой данных.
2.3.	Функциональные ограничения
•	ОС не ниже Windows 10;
•	Отсутствует возможности создания новых сущностей в БД и изменения атрибутов таблиц;
•	Для валидации данных требуется подключение к внешнему API.
 
3.	ОПИСАНИЕ ЛОГИЧЕСКОЙ СТРУКТУРЫ
3.1.	Алгоритм программы
1)	Запуск;
2)	Проверка заполнения полей;
3)	Поиск пользователя по логину;
4)	Проверка блокировки;
5)	Проверка пароля (3 попытки);
6)	Открытие формы CAPTCHA;
7)	Сборка пазла (3 попытки);
8)	Проверка правильности;
9)	Проверка прав доступа (IsAdmin);
10)	Открытие главного окна с кнопками и вкладками.
3.2.	Используемые методы
•	DbContext.SaveChanges() – сохранение изменений в БД;
•	DbSet.Load() – загрузка данных в память;
•	context.Users.FirstOrDefault() – поиск пользователя по логину;
•	HttpClient.GetAsync() – GET-запрос к API;
•	ReadFromJsonAsync<Message>() – десериализация ответа API;
•	Form.ShowDialog() – открытие модальных диалоговых окон;
•	PictureBox.DoDragDrop() – реализация drag-and-drop для CAPTCHA;
•	Regex.IsMatch() – проверка строки на соответствие шаблону; 
•	tabControl1.TabPages.Remove() – скрытие вкладки для обычных пользователей;
•	button.Visible = false – скрытие кнопок удаления/обновления для обычных пользователей.
3.3.	Структура программы
Основные компоненты (классы форм):
•	 AuthForm.cs – форма авторизации;
•	CaptchaForm.cs – форма для капчи;
•	TSForm.cs – форма для взаимодействия с API;
•	MainForm.cs – главное окно с таблицами.
Классы моделей (сущности):
•	User – пользователи;
•	Customer – заказчики;
•	Material – материалы;
•	Order – заказы;
•	OrdersItem – позиции заказа;
•	Product – продукция;
•	Specification – спецификации;
•	SpecificationItem – позиции спецификации.
3.4.	Связи программы с другими программами
•	Взаимодействие с сервером MySQL;
•	Взаимодействие с внешним API TransferSimulator.
